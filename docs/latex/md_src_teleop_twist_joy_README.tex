The purpose of this package is to provide a generic facility for tele-\/operating Twist-\/based ROS2 robots with a standard joystick. It converts joy messages to velocity commands.

This node provides no rate limiting or autorepeat functionality. It is expected that you take advantage of the features built into \href{https://index.ros.org/p/joy/github-ros-drivers-joystick_drivers/\#foxy}{\texttt{ joy}} for this.\hypertarget{md_src_teleop_twist_joy_README_autotoc_md2}{}\doxysubsection{Executables}\label{md_src_teleop_twist_joy_README_autotoc_md2}
The package comes with the {\ttfamily teleop\+\_\+node} that republishes {\ttfamily sensor\+\_\+msgs/msg/\+Joy} messages as scaled {\ttfamily geometry\+\_\+msgs/msg/\+Twist} messages.\hypertarget{md_src_teleop_twist_joy_README_autotoc_md3}{}\doxysubsection{Subscribed Topics}\label{md_src_teleop_twist_joy_README_autotoc_md3}

\hypertarget{md_src_teleop_twist_joy_README_autotoc_md7}{}\doxysubsection{Install}\label{md_src_teleop_twist_joy_README_autotoc_md7}
For most users building from source will not be required, execute {\ttfamily apt-\/get install ros-\/\texorpdfstring{$<$}{<}rosdistro\texorpdfstring{$>$}{>}-\/teleop-\/twist-\/joy} to install.\hypertarget{md_src_teleop_twist_joy_README_autotoc_md8}{}\doxysubsection{Run}\label{md_src_teleop_twist_joy_README_autotoc_md8}
A launch file has been provided which has three arguments which can be changed in the terminal or via your own launch file. To configure the node to match your joystick a config file can be used. There are several common ones provided in this package (atk3, ps3-\/holonomic, ps3, xbox, xd3), located here\+: \href{https://github.com/ros2/teleop_twist_joy/tree/eloquent/config}{\texttt{ https\+://github.\+com/ros2/teleop\+\_\+twist\+\_\+joy/tree/eloquent/config}}.

PS3 is default, to run for another config (e.\+g. xbox) use this\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{ros2 launch teleop\_twist\_joy teleop-\/launch.py joy\_config:='xbox'}

\end{DoxyCode}
